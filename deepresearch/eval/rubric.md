# DeepResearch 2.0 — 研报评分细则（Rubric）

100 分制，6 个维度。
**结论：没有 evidence 的维度上限 = 权重 × 0.6**（写 evaluation JSON 的 Agent 必须遵守）。

---

## 维度 1：来源质量 (Source Quality) — 权重 20

衡量引用的"含金量"。

| 子项 | 评分要点 |
|---|---|
| 一手 vs 二手 | 论文 / 官方 README > 第三方博客 / 二手综述 |
| 时效 | ≥ 2024 papers 视为新；2022–2023 视为可接受；< 2022 需要在概念页里有理由 |
| 影响力 | repos: stars ≥ 1k 加分；papers：出现在多个综述里加分 |
| 多样性 | 论文 / 博客 / 仓库三类来源都有命中 |

| 分段 | 标准 |
|---:|---|
| 18–20 | 80% 以上引用是高信任种子；三类来源齐全；至少 3 篇 2024+ |
| 14–17 | 60% 高信任；三类齐全；时效一般 |
| 10–13 | 来源单一（只有一类）或大量 < 2022 |
| 0–9   | 出现错引 / 失效链接 / 重复来源 |

## 维度 2：覆盖完整度 (Coverage) — 权重 20

衡量是否回答了 `topics/document-parsing.yml > research_questions` 中的 7 个问题。

| 命中数 | 分值 |
|---:|---:|
| 7/7 | 19–20 |
| 6/7 | 16–18 |
| 5/7 | 13–15 |
| 4/7 | 10–12 |
| ≤3   | ≤ 9 |

> auto_check.py 会基于关键词大致估计；最终由 Agent 主观确认是否真"答过"。

---

## 维度 3：引用可追溯性 (Traceability) — 权重 20

每一句**结论级**论断（"X 比 Y 好"、"达到 Z 准确率"、"主流方案是 W"）必须有引用。

定义：**结论级语句** = 包含数字 / 比较 / 主张 / 推荐 的句子。

| citation_ratio | 分段 |
|---:|---|
| ≥ 0.9 | 18–20 |
| 0.7–0.9 | 14–17 |
| 0.5–0.7 | 10–13 |
| < 0.5 | ≤ 9 |

`citation_ratio` 由 auto_check.py 给出（基于句子级正则 + 引用模式探测）。

---

## 维度 4：结构与连贯性 (Structure) — 权重 15

| 检查 | 扣分 |
|---|---|
| 章节标题不层级 / 跳级 | -2 |
| 同一术语两种叫法 | -2 |
| 重复段（>50% 字面雷同）≥ 1 处 | -3 |
| 引言未铺垫主题 / 结论未呼应引言 | -3 |
| 章节字数极不均衡（标准差 / 均值 > 0.7） | -2 |

满分 15。

---

## 维度 5：洞察与判断 (Insight) — 权重 15

衡量是否回答了"为什么 / 何时不适用 / trade-off"。

| 检查点 | 加分 |
|---|---:|
| 至少 3 处显式 trade-off（如 "X 在中文表现不如 Y 但吞吐高 3x"） | +5 |
| 至少 2 处反例 / 失败案例 | +4 |
| 至少 1 个 "我们不敢下的结论" 段落 | +3 |
| 至少 1 处对未来 6–12 月的判断（不是抄趋势） | +3 |

满分 15，缺一项扣对应分数。

---

## 维度 6：结论稳定性 (Stability) — 权重 10

| 检查 | 扣分 |
|---|---|
| 同一对象在不同章节给出**矛盾**结论（A vs B 谁好） | -3 |
| 引用数字在不同章节不一致（如 "92.3" vs "92"） | -2 |
| 没有"不敢下的结论"小节 | -2 |
| 章节最后的 take-away 与全文主旋律不一致 | -2 |

满分 10。

---

## 总分汇总

```
total = 维度1 + 维度2 + ... + 维度6
       (max 100)
```

**评级**：

| 总分 | 评级 |
|---:|---|
| ≥ 85 | 专家级（可对外发布） |
| 70–84 | 工作级（社区演示足够） |
| 55–69 | 草稿（需 1 轮修订） |
| < 55 | 不可发（流程出了问题） |

## Wiki vs Direct 的预期分布

按 DeepResearch 2.0 的设计预期：

| 维度 | Wiki 预期 | Direct 预期 | 主要差距来源 |
|---|---:|---:|---|
| 来源质量 | 17–20 | 12–16 | Wiki 强制 evidence 字段 |
| 覆盖 | 17–20 | 13–17 | Wiki 有 roadmap 强制覆盖 |
| 可追溯性 | 18–20 | 8–13 | **核心差距** |
| 结构 | 13–15 | 8–12 | Wiki 概念页强制结构化 |
| 洞察 | 11–14 | 4–9  | **第二大差距** |
| 稳定性 | 8–10 | 4–7 | Wiki lint 抓矛盾 |

**总分预期：Wiki 80–95，Direct 50–70。**
若 Direct ≥ Wiki，要回头看 Wiki 编译是否流于形式（最常见原因：没写 evidence）。
